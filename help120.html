A success guide to the first entry-level course in the Computer Science curriculum.

Intro to the Course

CMSCI 120 is the Introduction to Computer Science Course. It is a freshmen level course that aims to serve as entry point for all future computer science classes; teaching not only basic coding, but how to properly think and function like a computer scientist. The course covers basic programming, problem-solving methods and the development of lower complexity algorithms; with a focus on the coding language, Python.

Source: Mount St. Mary's Course Catalog

Thinking Like a Computer Scientist

One of the most prominent struggles students may face during this course is the adoption of a Computer Scientist's mindset. The ability to understand a problem and think of the best solution to said problem is much easier said than done. If nothing else, a computer scientist's mindset should be the primary takeaway from this course, as it's a skill that no advancements in technology can really replace.

Integrated Development Environments

In order to begin programming, the use of an IDE, or Integrated Development Environment, is required. There are many popular IDEs, each with their own sets of advantages or disadvantages. Some of the more popular choices for Python are listed below to experiment with. However, it is highly likely that each professor to teach this course will have their own preferred IDE, and it is advised that you follow that preference.

  • Thonny : A barebones IDE, popular for being open source and without any extra features. It’s downsides also happen to be said lack of features.
  • Visual Studio Code : One of the most popular IDEs in general. This IDE supports every known coding language through extensions, not dissimilar to that of browser extensions. The application is popular for a reason, and is almost guaranteed to be used by every Computer Science major during their undergraduate studies.
  • PyCharm : The most popular Python specific IDE. Recommended for its built in advanced code analysis, debugging, and database integration. As opposed to the other IDEs mentioned, PyCharm's downside comes from some features being paywalled behind a 'Professional Edition' of the application.

Debugging

Debugging is a skill, that any programmer worth their salt, will use actively during any application development. The skill is, in short, the process of finding errors in code, processing the output error message, and fixing said errors. While many LLM and AI based tools have reduced perceived need for this skill, it is still very much an integral part of the computer scientist's toolkit. Below are a few debugging approaches that can be helpful in the occurrence of malfunctioning code.

  • Isolate: If something's confusing, try to break the code down into its simplest parts that you can have.
  • Binary Search: Brute forcing the code, removing bits piece by piece until the code eventually runs. This method excels when nothing sticks out just by looking at the code itself.

Outside the Class

Depending on the professor, it is possible for the amount of work outside class time to vary greatly. The biggest piece of advice that can be given, especially in the age of LLMs and AI, is to simply not use them. When encountering a hard problem to solve, if you solve it by hand you'll remember the solution. However, if you give the problem to ChatGPT to solve instead, you don't gain this muscle memory and don't learn anything in the process.

It is important to acknowledge that while, yes you can in theory use AI to pass many of your classes; you'll only get as much out the classes as you put into them. It's important to experiment outside of class. If you want to build a videogame, then build one. If you want to design your own website, then do it.

Even if you get all of your knowledge and code from Google, you still learn and apply new knowledge; and all of this will eventually become second nature.

Resources

Below are a series of completely free resources that not only explain Python, but also help with processes such as adapting a Computer Scientist's mindset, or learning how to add to your resume as a newcomer in the tech industry.

Guides/Documentation
  • Python.org The original website for Python. Houses the download for the language as well as all official documentation.
  • Automate the Boring Stuff An e-book detailing not only Python, but computational thinking in realistic ways.
  • Python Notes for Professionals A more straightforward, buisness like approach to learning Python.
  • Python Full Course - Programming with Mosh A comprhensive video introduction to Python and computing in general. This youtuber is one of the most well known and recommended in online spaces.
Hands On Practice
  • LeetCodeAn intermediate to advanced challenge website, focused on practicing skills for job interviews and real world scenarios.
  • HackerRankAnother website focused on job readiness, but with a much ligher learning curve and less commercialization.
  • CodeWarsAn advanced online coding practice tool, utilized by a few clubs on campus.
External Courses
  • Open PythonA completely free to use, series of online practice courses for all levels of users.
  • Cisco Networking Academy - Python EssentialsA Cisco sponsered course that gives information about the language, as well as network programming.
  • Introduction to Computer Science - Harvard CS50A completely free to audit course from the Ivy League School of Harvard. Well known for it's broad coverage of topics and ease of access.
